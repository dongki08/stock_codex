# 뉴스·공시 관계 분석 설계

> 🧭 인덱스: [00-INDEX.md](../../00-INDEX.md) · 카테고리 설계 · 상태 ✅ 확정 · 코드기준 2026-06-04

## 목적

정형 데이터 파싱과 추천 가격 계산은 기존 룰 엔진이 담당한다. Codex는 당일 뉴스와 공시 사이의 관계, 상충 정보, 핵심 위험을 종목별 구조화 신호로 해석한다.

## 실행 흐름

1. 시장별 거래대금 상위 후보의 당일 뉴스와 공시를 수집한다.
2. 시장별 후보 전체를 한 프롬프트로 묶어 Codex를 한 번 호출한다.
3. Codex JSON 응답을 종목별 `context_relation_analysis` 행으로 저장한다.
4. 추천 feature 계산에서 기존 컨텍스트 점수 70%, 관계 분석 점수 30%로 혼합한다.
5. 프리오픈 Telegram 추천에 관계 분석 요약과 위험도를 표시한다.

## 운영 원칙

- KOSPI, KOSDAQ, NASDAQ, NYSE 기준 하루 최대 4회 호출한다.
- URL과 원문 전체는 프롬프트에 넣지 않고 제목·요약·공시 유형만 전달한다.
- 응답 실패 또는 JSON 오류 시 저장하지 않으며 기존 룰 점수만 사용한다.
- 백테스트는 기준일과 같은 날짜의 관계 분석만 조회해 미래 참조를 차단한다.
- AutoResearch의 가중치 제안은 기존처럼 백테스트 검증 후 반영한다.
